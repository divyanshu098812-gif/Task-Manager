# ⚡ Deployment Quick Reference

## 🎯 5-Minute Deployment

### Step 1: Setup Database (Railway - Recommended)
```
1. Go to railway.app
2. Login with GitHub
3. New Project → MySQL
4. Copy credentials from Variables tab
```

### Step 2: Push to GitHub
```bash
git add .
git commit -m "Deploy to Render"
git push origin main
```

### Step 3: Deploy on Render
```
1. Go to render.com
2. Sign in with GitHub
3. New → Web Service
4. Select your repository
5. Add environment variables (see below)
6. Deploy
```

---

## 🔑 Environment Variables for Render

Add these in Render Dashboard → Environment tab:

| Variable | Value | Where to Get |
|----------|-------|--------------|
| `FLASK_ENV` | `production` | Type manually |
| `SECRET_KEY` | Auto-generated | Leave as generated by Render |
| `DB_HOST` | `containers-us-west-xxx.railway.app` | Railway Variables → MYSQLHOST |
| `DB_PORT` | `3306` | Railway Variables → MYSQLPORT |
| `DB_USER` | `root` | Railway Variables → MYSQLUSER |
| `DB_PASSWORD` | `xxxxx` | Railway Variables → MYSQLPASSWORD |
| `DB_NAME` | `railway` | Railway Variables → MYSQLDATABASE |

---

## 📊 Railway Database Setup

### Get Credentials:
1. Railway Dashboard → Your MySQL Service
2. Click "Variables" tab
3. Copy values:
   - `MYSQLHOST` → Use as `DB_HOST`
   - `MYSQLPORT` → Use as `DB_PORT`
   - `MYSQLUSER` → Use as `DB_USER`
   - `MYSQLPASSWORD` → Use as `DB_PASSWORD`
   - `MYSQLDATABASE` → Use as `DB_NAME`

## 💾 Migrate Local Data (Optional)

### Export Local Database:
```bash
mysqldump -u root -p task_manager_db > backup.sql
```

### Import to Railway:
```bash
mysql -h <RAILWAY_HOST> -P 3306 -u root -p<PASSWORD> railway < backup.sql
```

Replace:
- `<RAILWAY_HOST>` with your Railway MYSQLHOST
- `<PASSWORD>` with your Railway MYSQLPASSWORD

## 🐛 Quick Troubleshooting

### Database Connection Error?
→ Double-check environment variables in Render
→ Verify Railway database is running
→ Check logs for specific error message

### App Won't Start?
→ Check Render logs for Python errors
→ Verify requirements.txt has all packages
→ Ensure runtime.txt has `python-3.9.18`

### 500 Error?
→ Check Render logs
→ Verify database tables were created
→ Test database connection separately
